Sistemas de banco de dados Os bancos de dados são fundamentais para sites dinâmicos, eles contêm todas as informações administrativas e os conteúdos do site. A base de dados é um componente fundamental de um site dinâmico, contém todas as informações necessárias para o funcionamento do site. Nesse contexto, o banco de dados deve conter informações administrativas como usuários, permissões e senhas, informações textuais, que é o conteúdo do site como tal, e outras informações que fazem parte do core do negócio. Banco de Dados Relacional Existem vários modelos de banco de dados, vamos citar um deles que é mais utilizado no desenvolvimento de sites, o modelo relacional. Isso tem fundamentos teóricos na álgebra relacional que foi proposta por Edgar &quot;Ted&quot; Codd na década de 1970, como uma espécie de teoria dos conjuntos aplicada a dados. Diagrama Entidade-Relacionamento Entidades Um dos conceitos importantes no modelo relacional é o conceito de entidade. Uma entidade representa algo significativo e contém atributos relevantes para o sistema. Por exemplo, se nosso site for um blog, uma possível entidade seria &quot;artigo&quot; e alguns atributos possíveis seriam sua &quot;data de publicação&quot; e seu &quot;título&quot;. Se nosso site for uma loja online, uma possível entidade seria &quot;produto&quot; com os atributos possíveis &quot;nome&quot; e &quot;preço&quot;. Chaves É importante que cada entidade tenha um identificador único conhecido como sua chave primária, que pode ser artificial, ou seja, gerada pelo sistema, ou natural, ou seja, normalmente faz parte da entidade. As chaves facilitam as buscas na base de dados e permitem estabelecer relações entre as entidades. Relacionamentos Um relacionamento é uma associação entre duas entidades, podemos identificar esses relacionamentos quando usamos verbos como tem, contém, faz parte, está inscrito, etc. Estabelecemos relacionamentos ao projetar o banco de dados e são baseados na experiência em nosso negócio. Por exemplo, se nosso site é um blog sabemos que um autor escreve vários artigos, então encontramos uma relação entre a entidade &quot;artigo&quot; e a entidade &quot;autor&quot;. Se o nosso site for uma loja online, sabemos que uma compra tem um ou mais produtos, pelo que encontramos uma relação entre a entidade &quot;compra&quot; e a entidade &quot;produto&quot;. Para implementar relacionamentos, o conceito de chave estrangeira é usado. A chave estrangeira é uma chave primária para uma entidade e é copiada como atributo de outra entidade, estabelecendo assim o relacionamento. Seguindo o exemplo do blog, para relacionar um autor aos seus artigos adicionaremos um novo atributo &quot;escrito por&quot; ao &quot;artigo&quot;, que conterá o valor da chave primária da entidade &quot;autor&quot;. Pesquisas Um dos benefícios de ter um modelo matemático para bancos de dados relacionais é que facilitou a criação de uma linguagem de pesquisa estruturada conhecida como Structured Query Language (SQL). Através disso podemos executar atualizações e consultas nos dados. Por exemplo, atualizar atributos de alguma entidade, procurar entidades que atendam a determinadas condições ou localizar grupos de entidades associadas, entre muitas outras opções. Implementação computacional Os sistemas de gerenciamento de banco de dados relacional, RDBMS por sua sigla em inglês, implementam o modelo relacional através do conceito de tabela. Uma tabela contém linhas e colunas. Cada entidade é registrada em uma linha da tabela e cada um de seus atributos é registrado em cada coluna da linha. Por fim, as tabelas se materializam como arquivos em disco, para que suas informações sejam gravadas de forma permanente e duradoura. Exemplo de tabela &quot;AUTHOR&quot; Além disso, esses sistemas oferecem funcionalidades como gerenciamento de usuários e permissões, acesso simultâneo pela rede, criação de backups e análise de desempenho, entre outros. No mercado encontramos muitos desses sistemas, podemos citar MySQL, Oracle e PostgreSQL entre os mais conhecidos. Existem diferentes critérios para classificá-los, uma infinidade de implementações e inúmeros drivers para se conectar a eles a partir de várias linguagens de programação. Dependendo da complexidade do site, podemos ter um ou mais bancos de dados, inclusive é possível misturar diferentes tipos de bancos de dados no mesmo projeto de acordo com as necessidades. Entre as alternativas ao modelo relacional, podemos encontrar outras formas de armazenamento, como repositórios de valores-chave, bancos de dados orientados a objetos, tabelas distribuídas, bancos de dados orientados a documentos ou armazenamento baseado em log. Cada um deles tem semelhanças e diferenças com o modelo relacional. Podemos citar MongoDB, Ozone DB, Oracle Berkeley Database, Apache Cassandra e Hypertable entre outros.
