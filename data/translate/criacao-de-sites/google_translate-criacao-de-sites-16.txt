Tecnologias e conceitos no backend web O backend engloba tudo o que roda no servidor, onde são tratadas as tecnologias e conceitos que permitem a criação de sites dinâmicos. No início da década de 1990, foi criado o primeiro mecanismo para um servidor web responder com conteúdo dinâmico: CGI. A especificação Common Gateway Interface permite que um servidor web estático delegue a outro programa a geração de conteúdo dinâmico e então receba a resposta e redirecione-a para o navegador. Nos anos seguintes outras abordagens foram utilizadas, alguns servidores web adicionaram módulos para poder executar código sem o uso de CGI, outras tecnologias permitiram misturar instruções de programação com tags html e servidores especificamente projetados para gerar conteúdo dinâmico também foram criados. Sem entrar nos detalhes de cada linguagem ou plataforma, podemos falar em termos gerais sobre como funciona um site dinâmico, lembre-se que para cada solicitação do navegador há uma resposta do servidor. Um servidor web dinâmico expõe a solicitação e resposta http, o que significa, em palavras mais simples, que temos acesso de nosso código a essas duas estruturas na forma de uma estrutura de dados ou como um objeto. Solicitação HTTP A solicitação http contém vários valores que são úteis para saber qual conteúdo devemos gerar ou qual ação devemos executar, um dos mais úteis são os parâmetros, são uma série de nomes e valores em forma de texto. Dependendo do método da solicitação, esses valores podem ser visíveis no final da url ou não visíveis no corpo da solicitação. Em qualquer caso, podemos acessar esses nomes e valores do nosso código. Resposta HTTP O corpo da resposta http varia dependendo do conteúdo, pode ser texto simples representando folhas de estilo, tags html ou código javascript, mas também pode ser conteúdo binário, como imagens ou documentos. O navegador poderá informar o tipo de conteúdo lendo o valor do cabeçalho Content-Type, portanto, precisamos definir o valor correto do nosso código de back-end. Sessão HTTP A comunicação em um ambiente web é uma comunicação stateless (stateless) isto significa que o servidor web não sabe se houve ou haverá uma comunicação com o mesmo cliente, ele simplesmente responde a cada requisição como se fosse uma nova e isolada solicitar. No contexto de um site dinâmico, é necessário manter um estado, ou seja, poder saber quem está fazendo a solicitação e quais solicitações foram feitas anteriormente. Para conseguir isso, o conceito de uma sessão é usado. Em termos informáticos, uma sessão é um período de tempo em que uma série de mensagens são passadas entre duas máquinas.Em um site dinâmico, uma sessão corresponde a um grupo de solicitações http e respostas de um usuário. O conceito de sessão é importante porque permite ao servidor distinguir o usuário que faz as solicitações para gerar o conteúdo apropriado, por exemplo, por segurança não podemos mostrar opções administrativas para todos os usuários, apenas para aqueles que forneceram um nome de usuário e senha válidos. A sessão http geralmente é implementada gerando um código identificador único por usuário que é armazenado no navegador através de um cookie, o navegador sempre envia esse identificador em todas as solicitações para o mesmo site enquanto o servidor web armazena o identificador e associa alguns dados importantes a reconhecer o usuário. Código, templates e dados no web backend No frontend não temos muitas opções em termos de linguagens de programação, até o momento só temos JavaScript (embora possamos usar indiretamente outras linguagens que sejam capazes de produzir código js). Pelo contrário, no backend temos várias opções, podemos escrever a lógica do nosso site dinâmico em várias linguagens e plataformas que veremos mais adiante. A partir do código backend temos acesso a duas coisas importantes no desenvolvimento de sites dinâmicos: os templates e os dados. O sistema de templates de backend é a tecnologia que nos permite substituir apenas os dados dinâmicos em um documento html. Depende um pouco da linguagem ou plataforma que usamos, mas em geral consiste em um arquivo de texto com determinadas especificações ao qual são aplicados alguns dados, gerando assim um novo documento. Esses dados que mencionamos geralmente vêm de um ou mais bancos de dados. Os bancos de dados, que veremos mais adiante, desempenham um papel muito importante no desenvolvimento de sites dinâmicos, pois contêm as informações necessárias para gerenciar e gerar nosso conteúdo dinâmico.
